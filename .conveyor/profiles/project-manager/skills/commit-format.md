# Commit Message Format

```
<type>: <short subject>

<body (optional)>
```

Types:
- `feat` ‚Äî new feature
- `fix` ‚Äî bug fix
- `refactor` ‚Äî code change without feature/fix
- `docs` ‚Äî documentation only
- `chore` ‚Äî tooling, config, dependencies
- `test` ‚Äî adding/updating tests

Examples:
- `feat: add sync-daemon retry logic`
- `fix: handle null task description in reviewer`
- `refactor: extract retry helper to shared module`
